# Headwind v0.2.0-alpha

**Release Date:** January 2025
**Status:** Alpha (Core functionality complete, comprehensive testing in progress)

This release represents a major milestone with core operator functionality complete and extensive new features.

## üéâ Major Features

### Multi-Workload Support
- ‚úÖ **Deployments** - Full support with approval workflow
- ‚úÖ **StatefulSets** - Complete implementation for stateful applications
- ‚úÖ **DaemonSets** - Node-level service support
- ‚úÖ **Flux HelmReleases** - Automatic chart version discovery

### Helm Chart Auto-Discovery
- Automatically queries Helm repositories for new chart versions
- Supports both HTTP and OCI Helm repositories
- Private repository authentication via secretRef
- Direct updates without approval (configurable)
- Full metrics and notification integration

### Rollback Capabilities
- **Automatic Rollback**: Monitors deployment health and reverts on failures
- **Manual Rollback**: API and kubectl plugin support
- Update history tracking (last 10 updates per container)
- Health check-based failure detection

### Notification System
- **Slack Integration**: Rich Block Kit formatting
- **Microsoft Teams**: Adaptive Cards support
- **Generic Webhooks**: HMAC signature verification
- Multiple channel support simultaneously
- Event notifications: updates, approvals, rejections, rollbacks

### Registry Polling
- Dual detection: digest changes + new version discovery
- Helm chart polling for OCI and HTTP repositories
- Private registry authentication
- Smart filtering and deduplication
- Configurable polling intervals

## üîß Features

### Update Policies
- `none` - Never update (default)
- `patch` - Only patch versions (1.2.3 ‚Üí 1.2.4)
- `minor` - Patch and minor (1.2.3 ‚Üí 1.3.0)
- `major` - All versions including major
- `all` - Any new version
- `glob` - Pattern matching (e.g., `v1.*-stable`)
- `force` - Force update regardless of version

### Approval Workflow
- Full HTTP API for approval/rejection
- UpdateRequest CRDs for state management
- kubectl plugin for CLI operations
- Auto-cleanup support via CronJobs
- Bulk approval workflows

### kubectl Plugin
Complete CLI for:
- Listing pending updates
- Approving/rejecting updates
- Viewing update history
- Rolling back deployments

## üöß Known Limitations

**OCI Helm Repositories**: Due to `oci-distribution` crate v0.11 limitation, OCI Helm repositories may query Docker Hub when chart names match common Docker images. Workaround: Use HTTP Helm repositories (fully supported).
